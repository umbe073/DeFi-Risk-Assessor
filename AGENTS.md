# Agents

## Cursor Cloud specific instructions

This is a Python CLI tool (no web server, no database, no Docker). See `README.md` for product overview.

### Running the tool

- **Main assessment**: `python3 defi_complete_risk_assessment.py` — processes tokens from `tokens.csv`, outputs `risk_report.json` and `risk_report.csv`. Takes ~45 seconds with placeholder/fallback API keys.
- **Excel post-processor**: `python3 update_risk_assessment_xlsx.py` — updates an existing `DeFi Tokens Risk Assessment Results.xlsx` with data from `risk_report.json`. Requires the `.xlsx` file to already exist (it is not auto-generated by the main script).
- **Shell launcher**: `bash run_risk_assessment.sh` — runs both scripts in sequence.

### API keys

The tool requires a `.env` file in the project root. At minimum `INFURA_API_KEY` and `ETHERSCAN_API_KEY` are needed for real data. Without real keys, the tool still runs using fallback/cached data and free APIs (CoinGecko, CoinPaprika, DeFi Llama). See the docstring at the top of `defi_complete_risk_assessment.py` for the full list of optional API keys.

### Linting

No project-specific linter config exists. Use `flake8 --max-line-length=200` for style checks. Pre-existing warnings (bare excepts, trailing whitespace, etc.) are expected in the current codebase.

### Testing

No automated test suite exists. Manual verification is done by running `python3 defi_complete_risk_assessment.py` and checking the generated `risk_report.json` output.

### Key caveats

- The `osascript` completion dialog in `working_progress_bar.py` is macOS-only; it logs a harmless `FileNotFoundError` on Linux — this is expected.
- The main script uses `DATA_DIR` with a `data/` subdirectory fallback to the project root. In the current repo layout, all data files live at the project root.
- Output files (`risk_report.csv`, `risk_report.json`, `logs/`) are generated in `DATA_DIR`.
