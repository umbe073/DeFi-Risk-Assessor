FAQ

Most common login, subscription, and risk-assessment questions with direct fixes.

Account and Access

I cannot log in even with the correct password. What should I check first?

Confirm you are using the correct email account, check for browser auto-fill mistakes, and verify caps-lock is off. If 2FA is enabled, password-only login will not complete until you enter the authenticator code.

My 2FA code is always rejected.

Sync your phone time automatically and retry with a fresh code. TOTP is time-based, so device clock drift is the most common reason for failures.

I created an account but did not receive a confirmation code email.

Use the Send Confirmation Code button on the verification page, then check spam/junk. The code expires in 30 minutes. Never share your code with anyone.

How long is my email confirmation code valid?

30 minutes from the moment it is generated. After expiration, request a new code from the same verification page.

Risk Assessment Output

Why do I see N/A fields in the report?

N/A usually means the upstream data provider did not return that metric for the selected token/chain at request time. Retry after a few minutes; if it persists, open a support ticket with token address and chain ID.

Why are some sections returning 0 results?

Zero values can be legitimate for low-activity tokens, newly deployed contracts, or unavailable historical windows. Cross-check with run timestamp and API health before treating it as a tool failure.

What does "unreachable API" mean in live assessment logs?

At least one provider endpoint timed out, refused connection, or returned an error status. The run continues with partial data when possible, but final confidence may be lower.

I suspect a fake red-flag in the report. What should I do?

Open a ticket and include token address, chain, run UTC timestamp, and the exact flagged section. Our team can replay the run and validate whether the flag is data-noise or a true risk signal.

Billing and Subscriptions

Do I receive an email when my signup is completed?

Yes. After successful email-code verification, a welcome confirmation email is sent from no_reply@hodler-suite.com.

I paid but my subscription access is not active yet.

Payment and entitlement sync can take a short processing window. If access is still locked after confirmation, send your payment reference and account email via Help Center.

Support

What should I include in a support ticket for faster resolution?

Provide your account email, exact issue, token address (if relevant), chain/network, UTC timestamp, and screenshots/log excerpts. This significantly reduces triage time.

How do I track my ticket?

Every request receives a unique ID (format: HD-YYYYMMDD-HHMMSSffffff-XXXXXX) in the confirmation message. Include that ID in follow-up communication.
